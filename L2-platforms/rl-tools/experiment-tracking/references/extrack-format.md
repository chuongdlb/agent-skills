# ExTrack Format Specification

## Directory Layout

```
experiments/
  <timestamp>/                          # ISO-ish: 2024-01-15_14-30-00
    <hash>_<name>/                      # Short hash + experiment name
      <env>_<algo>/                     # Environment + algorithm
        <seed>/                         # Integer seed
          steps/
            <step>/                     # Training step number
              checkpoint.h5             # HDF5 model weights
              trajectories.json.gz      # Compressed trajectories
          return.json                   # Evaluation returns
          ui.esm.js                     # Render function
          description.txt               # Description
```

## return.json

```json
{
  "steps": [100, 200, 300, 400, 500],
  "returns": [-180.5, -120.3, -75.2, -40.1, -15.7]
}
```

- `steps`: Array of training step numbers where evaluation occurred
- `returns`: Mean episode return at each evaluation point

## trajectories.json.gz

Gzip-compressed JSON array. Each element is an episode:

```json
[
  {
    "parameters": { ... },
    "states": [
      {"theta": 1.5, "theta_dot": 0.2},
      {"theta": 1.3, "theta_dot": -0.1}
    ],
    "actions": [[0.5], [-0.3]],
    "rewards": [-2.1, -1.8]
  }
]
```

## ui.esm.js

ES6 module with two exports:
```javascript
export async function init(canvas, options) { ... }
export async function render(ui_state, parameters, state, action) { ... }
```

Generated by the environment's `get_ui()` function. The ExTrack UI library loads this module dynamically to render trajectory replays.

## checkpoint.h5

HDF5 file structure:
```
/actor/
  layers/
    0/
      weights  [HIDDEN_DIM x INPUT_DIM]
      biases   [HIDDEN_DIM]
    1/
      weights  [HIDDEN_DIM x HIDDEN_DIM]
      biases   [HIDDEN_DIM]
    2/
      weights  [OUTPUT_DIM x HIDDEN_DIM]
      biases   [OUTPUT_DIM]
/critic_1/
  ...
/critic_2/
  ...
```
